# Explain Like Architect

When asked to explain:
1. Start with what problem this feature solves.
2. Then explain file roles.
3. Then explain data flow.
4. Then explain the smallest important code fragment.
5. End with:
   - what is foundational
   - what is temporary
   - what the next layer likely is

## Style
- Use simple Russian.
- Avoid jargon where possible.
- Do not explain line by line first.
- Prefer architecture and meaning over syntax trivia.
