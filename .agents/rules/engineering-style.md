# Engineering Style

## Core rules
- Prefer the simplest working solution.
- Prefer modular files with one clear responsibility.
- Avoid clever abstractions unless necessary.
- Keep functions small and readable.
- Do not create large monolithic files unless explicitly requested.

## Required after each implementation
Always provide:
1. changed files
2. one-sentence role of each file
3. where state lives
4. where interaction logic lives
5. what is temporary and what is foundational
