# Feature: Manual Edge Creation

## Goal
Позволить пользователю создавать рёбра (связи) между двумя существующими узлами, используя простой пошаговый процесс в DOM-интерфейсе: сначала выбирается исходный узел, нажимается кнопка старта связи, затем выбирается целевой узел.

## Acceptance Criteria
- В `App.tsx` добавлено новое состояние `linkingSourceId` (хранит `id` начального узла).
- В `NodeCard` добавлена кнопка «Start Link» (если режим связи неактивен) и «Cancel Link» (если активен).
- При клике на другой узел во время активного режима связи создается новое ребро в `edges`.
- Запрещено создание self-links (когда исходный и целевой узел совпадают).
- Запрещено создание дубликатов рёбер (проверяем связь в обоих направлениях).
- После успешного создания или отмены (по кнопке или при клике в пустой фон сцены), режим связи сбрасывается.
- Архитектура методов создания связи должна быть готова к будущему внедрению системы drag-to-connect.

## Files / Modules Affected
- **`src/App.tsx`**:
  - `const [edges, setEdges] = useState<EdgeData[]>(INITIAL_EDGES);` (добавляем `setEdges`).
  - `const [linkingSourceId, setLinkingSourceId] = useState<string | null>(null);`.
  - Логика обработчика `handleNodeSequence`: перехватывает выбор узла, если активен `linkingSourceId`.
- **`src/ui/NodeCard.tsx`**:
  - Принимает пропсы `isLinking: boolean`, `onStartLink: () => void`, `onCancelLink: () => void`.
  - Отрисовывает кнопку старта или отмены под/над формой редактирования.

## Risks
- Перехват кликов: нужно убедиться, что клик по второму узлу не просто "снимает выделение" с первого, а правильно триггерит создание ребра. Использование единой функции `handleNodeInteraction(id)` должно решить этот конфликт чисто.

## What will NOT be included
- Визуальное рисование "временного ребра" (мышки), тянущегося за курсором при связывании.
- Удаление рёбер.
- Лейблы (названия) на рёбрах.
- Сохранение между перезагрузками.
- Бэкенд.

## Extensibility Note: Future Drag-to-Connect
Дизайн оставляет ядро связывания расширяемым. Мы разделяем UX-вызов и саму логику.
Сейчас UX-вызов — это кнопка «Start Link» (вызывает функцию `startLink(id)`) и последующий клик на узел (вызывает `completeLink(id)` внутри координатора).
В будущем, чтобы добавить Drag-to-Connect, мы не будем менять эти функции. Нам нужно будет лишь добавить событие `onPointerDown` (drag start) на 3D-сферу, которое точно так же дернет `startLink(id)`, и событие `onPointerUp` (drop), которое дернет `completeLink(id)`. Вся логика валидации (без дубликатов, без само-связей) останется нетронутой в `App.tsx`.
