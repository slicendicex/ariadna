# Выделение компонента Node3D

Мы превращаем хардкодную тестовую сферу в полноценный компонент архитектуры `Node3D`, принадлежащий слою отрисовки сцены. Это первый шаг к модульности, где логика визуализации узлов графа отделяется от конфигурации самой сцены.

## Цель (Goal)
Извлечь `<TestSphere />` из файла `Scene.tsx` в свой собственный файл с понятной зоной ответственности. Визуальное поведение остается неизменным.

## Критерии приемки (Acceptance Criteria)
- Компонент `<TestSphere />` удален из `Scene.tsx`.
- Создан новый файл `src/scene/Node3D.tsx`, содержащий логику отрисовки и анимации узла.
- Приложение запускается без изменения визуального поведения (сфера вращается, камера управляется мышью).
- Имя файла отражает его специфику в 3D слое (не просто `Node`).

## Затронутые файлы/модули
- `src/scene/Node3D.tsx` `[NEW]` — компонент для отрисовки единичного узла графа.
- `src/Scene.tsx` `[MODIFIED]` — рендерит `<Node3D />` вместо своей хардкодной реализации.

## Зависимости
- **Новых зависимостей не требуется**. Используем текущий стек.

## Что НЕ включено
- Обработка событий клика мыши.
- Всплывающие 2D карточки (карточки узлов).
- Перетаскивание объектов (драг-н-дроп) и создание связей между ними.
