# Рендеринг множества узлов (Data-Driven Scene)

Мы переходим от жесткой вёрстки одного компонента к data-driven подходу (отрисовке на основе данных). Вместо одного хардкодного вызова `Node3D`, сцена будет отрисовывать массив конфигурационных данных `MOCK_NODE_DATA`. 

## Цель (Goal)
Определить базовый массив данных `MOCK_NODE_DATA` внутри файла сцены (на уровне модуля), и рисовать узлы динамически через `Array.map()`. Это фундаментальный шаг к интеграции полноценного State-менеджера в будущем.

## Критерии приемки (Acceptance Criteria)
- В файле `src/Scene.tsx` заведен локальный константный массив `MOCK_NODE_DATA` вне компонента, на уровне модуля.
- Элемент массива содержит только минимальные поля: `id`, `position`, `color` (без метаданных и связей).
- Логика `<Canvas>` использует метод `.map()` для генерации списка компонентов `<Node3D key={node.id} ... />`.
- В браузере визуально отображаются несколько вращающихся сфер.

## Затронутые файлы/модули
- `src/Scene.tsx` `[MODIFIED]` — внедрение `MOCK_NODE_DATA` и `.map()` обвязки.
- `src/scene/Node3D.tsx` — без изменений.

## Зависимости
- **Без новых зависимостей**. 

## Что НЕ включено
- Глобальный State Management (состояние остается статической константой вне дерева React).
- Выделение по клику (click selection) и лейблы.
- Связи (edges).
