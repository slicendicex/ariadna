# Feature: Edge Deletion

## Goal
Позволить пользователю удалять рёбра, связанные с выделенным узлом, как по одному (кликая по целевому узлу), так и массово (удаляя все связи выделенного узла разом). Перевести стейт взаимодействия из узкоспециализированного `linkingSourceId` в обобщенный `interactionMode` для масштабируемости.

## Acceptance Criteria
- В `App.tsx` состояние `linkingSourceId` заменяется на `interactionMode` (типа `'none' | 'link' | 'delete_one'`). Исходным узлом для операции теперь всегда считается `selectedNodeId`.
- В `App.tsx` обновлен `handleNodeInteraction(id)`:
  - Если активен режим `link`, клик по другому узлу создает связь (как и раньше).
  - Если активен режим `delete_one`, клик по другому узлу ищет ребро между ними и удаляет его (если оно есть).
  - В обоих случаях после действия режим сбрасывается в `'none'`, а фокус переключается на целевой узел.
  - Клик в пустой фон сбрасывает и выделение узла, и режим взаимодействия.
- В `App.tsx` добавлена функция `deleteAllLinks()`: отфильтровывает все рёбра из `edges`, где исходный узел участвует как `fromId` или `toId`.
- В `NodeCard` удалены старые пропсы старта/отмены связи. Вместо них прокидываются `interactionMode`, `onSetInteractionMode` и `onDeleteAllLinks`.
- В `NodeCard` отображаются 3 кнопки: 
  - "Start Link" / "Cancel Link" (включает/выключает режим 'link').
  - "Delete Link" / "Cancel Delete" (включает/выключает режим 'delete_one').
  - "Delete All Links" (мгновенно вызывает функцию).
- Исключены действия с самим собой (клик по тому же узлу в активном режиме будет проигнорирован с последующим сбросом режима).

## Files / Modules Affected
- **`src/App.tsx`**:
  - Замена `linkingSourceId` на `interactionMode`.
  - Изменение логики внутри `handleNodeInteraction`.
  - Добавление `deleteAllLinks`.
- **`src/ui/NodeCard.tsx`**:
  - Обновление интерфейса `NodeCardProps`.
  - Рендер трех кнопок управления связями внизу карточки (с соответствующими цветами: голубой для "Start", жёлтый/оранжевый для "Delete", красный для "Delete All").

## Risks
- Перегрузка карточки интерфейсом: теперь в ней 3 дополнительные кнопки. Мы разместим их компактно, чтобы они не отвлекали от редактирования содержимого узла.
- Поведение `selectedNodeId`: благодаря унификации взаимодействия, выделение целевого узла в `App.tsx` автоматически и бесшовно переключит фокус на него и сбросит `interactionMode`, что предотвратит баги состояния.

## What will NOT be included
- Выделение самих рёбер в пространстве 3D (кликаем только по самим узлам-сферам).
- Модалки подтверждения (типа "Вы уверены?").
- Лейблы на рёбрах.
- Сохранение в бэкенд.

## Extensibility Note
Как и создание связей, удаление работает через связку UX-вызова модификатора (`interactionMode = 'delete_one'`) и последующего "дропа" (`handleNodeInteraction(target)`). В будущем, если мы добавим, например, инструмент "красный лазер" (тянущийся за мышкой), он сможет просто включить этот `interactionMode` и позволить пользователю "перерезать" связи дропом на узел. Логика графа останется нетронутой в `App.tsx`.
