# Конфигурация компонента Node3D через Props

Мы сделаем наш текущий `Node3D` переиспользуемым компонентом. Вместо хардкода цвета и позиции внутри самого файла, мы вынесем их в явные входные данные (props) TypeScript интерфейса.

## Цель (Goal)
Описать интерфейс `Node3DProps` и научить сцену передавать параметры внутрь узла. Это абстракция, позволяющая `Scene` управлять тем, "каким" должен быть узел, в то время как `Node3D` управляет тем, "как" он рисуется.

## Критерии приемки (Acceptance Criteria)
- В `src/scene/Node3D.tsx` описан интерфейс `Node3DProps` со свойствами `position` и `color`. Оба свойства опциональны.
- Если свойства не переданы, компонент использует значения по умолчанию (`position=[0,0,0]`, `color="orange"`).
- `Node3D` применяет переданные параметры к `mesh.position` и `material.color`.
- В `src/Scene.tsx` происходит явная передача параметров компоненту `<Node3D color="orange" position={[0, 0, 0]} />`.

## Затронутые файлы/модули
- `src/scene/Node3D.tsx` `[MODIFIED]` — внедрение TypeScript интерфейса и применение пропсов.
- `src/Scene.tsx` `[MODIFIED]` — передача пропсов сверху вниз.

## Ограничения и зависимости
- **Без новых зависимостей**.
- Только рефакторинг подпиток (props).

## Что НЕ включено
- Выделение по клику (click selection).
- Карточки (card) текста.
- Перетаскивание (drag) и связи (edges).
